How to use grep and piping.


So you ever used ls in you command line and suddenly got smacked in the face with screenfuls of data? Piping, filtering and especially Grep are here to help. When you type text into the command line and press enter, you are using your operating system's standard input, which is a really nice way to interact with the computer. What is placed back on the screen is standard output, also heavily tied to your operating system. Now you can actually direct this output into files, filters and even programs using some pretty cool operators.


> redirect operator

rob$ ls > file.txt (this places the output of ls into a text document named file)

However, if the file already existed then you are in deep trouble, because all of its previous contents are erased. To combat this, you can use the append operator

>> append operator

rob$ ls >> file.txt (This appends the output of ls into a text document named file)

| Piping operator

rob$ ls | sort (this sorts and then prints ls output alphabetically using the filer sort!)


Now more on piping so that we can use some filters! Some common filters in UNIX are:

sort (sorts alphabetically)

grep (searches for given pattern)

wc (counts the words lines and characters)


examples

rob$ ls | sort (sorts ls output alphabetically)

rob$ ls | grep .txt (lists all files with .txt extension in current directory)

rob$ ls | grep .txt | sort (lists all .txt files alphabetically!)

rob$ ls | wc (lists the lines words and characters in ls output... Or the number of files!)

rob$ ls | grep .txt | wc (lists the number .txt files in the current directory)


As you can see, you can pipe standard output through multiple filters, allowing you to get some very specific stuff.


Now finally, if you want to filter through the actual contents of a file, use the the commands "more" or "cat". These commands put the contents of a file onto the terminal screen. But be careful with these because and redirection because you can accidentally overwrite some important files.


Examples

rob$ more mydocument.txt | grep Author

This would go inside the mydocument.txt file and output the line which contains "Author"
